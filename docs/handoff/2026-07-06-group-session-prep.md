## 次にやること（優先順）

1. **共有方法を決める。** 3つの選択肢（文章のみ／図解／その他）を、目的（Slack投稿＋画面共有＋口頭発表の補助）に照らして相談する。参考観点：
   - 画面共有しながら3分で口頭説明する前提なので、情報量を絞った視覚的資料の方が伝わりやすい可能性がある（`creating-visual-explainers`スキルは`personal-visual-explainers`ではなくこのリポジトリの`output/`から直接使える）
   - Slackへの投稿もするので、テキストでもそのまま読めるコピー可能な形（Markdownまたはスクリーンショット）も必要になる
   - ステップ⑥の最終図解と共有できる部分があるかどうかも検討する（完全に同じにするか、グループセッション用は簡易版にするか）
2. **共有する内容3点を整理する**（`docs/assignment.md`の「共有する内容」節・添付スクリーンショットと同内容）：
   - 自分はどんな方針でスキルを自分専用にしようとしているか → ステップ④の3層構造の設計（このファイルの「完了したこと」参照）

   - すでにスキルを動かしてみた人は、出てきた文章と次に直したい点（任意）→ `output/before-tuning.md`・`output/after-tuning.md`の比較
3. **1人3分の発言原稿を作成する。** 発表3分＋フィードバック5分の目安に収まる長さで、上記3点を簡潔に話せる原稿にする
4. （その後）ステップ⑥：月次課題最終提出用の図解をこのリポジトリの`output/`に作成し、Surgeにデプロイして公開URLを取得する
