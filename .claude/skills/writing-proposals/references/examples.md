# 提案文書の模範例

意思決定者の問いに、結論ファーストで答える提案文書の例。

## 目次

- 対話の流れ（例）
- 例0：Before / After（上流を埋める効果）
- メンツを守る言い換え（NG / OK）
- 例1：在宅勤務を週2日に増やす
- 例2：採用管理ツールの導入（他ツール比較あり）
- 例3：ノー残業デーの導入
- 例4：先延ばしにされがちな相手に、日時を決めさせるLINE提案文
- 例5：期日を聞いても返ってこない相手に、こちらが期日を設定して無返信でも進めるLINE提案文

---

## 対話の流れ（例）

ユーザーが雑に投げ、AIが1問ずつ推奨案つきで材料を引き出す。

> ユーザー：定例会議を減らしたい
> AI：①は「定例会議を減らす」ですね。②決めるのは誰ですか？ チームの定例なら直属の部長が承認者になりがちです。これでいいですか？
> ユーザー：部長です
> AI：③部長の頭の中の問いは「どの会議か／何がよくなるか／自分は何をすればいいか」が刺さります。この3つに答える形でOKですか？
> ユーザー：OK
> AI：④根拠になる数字はありますか？（なければ体験談でも可）
> ユーザー：週7本・12時間、直近は7人中2人しか発言していない
> AI：十分です。提案文書を作ります。

材料が揃うと、下の「例0：After」のような文書が出力される。

## 例0：Before / After（上流を埋める効果）

同じ「定例会議を減らしたい」でも、上流（誰に・相手の問い・何を言い切るか）を埋めるかどうかで結果が変わる。

**Before（AIに丸投げ・上流が空っぽ）**

```
お疲れ様です。会議運営の効率化についてご提案がございます。
最近、会議時間の増加が生産性低下の要因として指摘されています。
つきましては、定例会議の頻度の見直しをご検討いただけますと幸いです。
```

丁寧だが、どの会議か・何がよくなるか・何をしてほしいかに答えていない。意思決定者は「それで？」となる。

**After（上流を埋めてから書く）**

```
【提案】報告だけの定例会議3本を隔週にして、浮いた月6時間を提案準備に充てます。ご承認をお願いします。

◎ 現状の課題
定例会議は週7本・延べ12時間。うち報告だけの3本が週3時間を占める。
直近の会議では発言者が7人中2人だった。

◎ 提案する理由
報告だけの会議は資料の前日共有で代替できる。議論のある会議は残す。

◎ リスクと対策
共有漏れを防ぐため、資料は前日17:00までに投稿するルールにする。
```

## メンツを守る言い換え（NG / OK）

同じ提案でも、過去を否定する書き方は通りにくい。状況の変化として語り、相手の功績を立てる。

**NG（過去を否定）**

```
今の定例会議は無駄が多く非効率です。やめるべきです。
```

**OK（状況変化＋功績を立てる）**

```
定例会議は立ち上げ期に足並みを揃えるのに役立ってきました。
人数も増えて自律的に動ける今、形を変えるタイミングだと思います。
まず1ヶ月、報告は資料共有に切り替えて試させてください。
```

## 例1：在宅勤務を週2日に増やす

```
【提案】在宅勤務を週2日に増やすことを承認してください。来月から3か月の試験運用にしたいです。

◎ 現状の課題
通勤に往復2時間かかり、集中して作業する時間を確保しにくい。
資料作成は1人で進める作業が多く、出社の必要性が低い。

◎ 提案する理由
移動時間を作業に充てられ、締め切り前の残業を減らせる。
週2日に絞れば、対面が必要な会議や相談は出社日にまとめられる。

◎ リスクと対策
連携が滞らないよう、出社日を火・木に固定してチームで揃える。
3か月後に稼働と成果を振り返り、継続するか判断する。
```

## 例2：採用管理ツールの導入（他ツール比較あり）

```
【提案】採用管理ツールをHRtable（月額3万円）で導入したいです。
Googleスプレッドシートで管理している採用フローを移行する承認をお願いします。

◎ 現状の課題
選考状況の共有にメール往復が発生し、1件あたり平均30分かかっている。
スプレッドシートの更新漏れで、面接の重複が月2〜3件発生している。

◎ 他ツールとの比較検討
・HRtable（月額3万円）— 採用サイトとAPI連携できる
・Recruit Board（月額2万円）— API連携できないため見送り
→推奨：HRtable（連携で更新漏れと面接重複を防げる）

◎ コスト対効果
月額3万円に対し、現在の対応工数（月約20時間）を半減できる見込み。
```

## 例3：ノー残業デーの導入

```
【提案】毎週水曜日をノー残業デーにしたいです。
定時の18時退勤を原則とすることを承認していただきたいです。

◎ 現状の課題
・定時以降は集中力が落ち、同じ作業でも時間がかかる
・平均残業時間は月40時間で、その時間帯の生産性は低い

◎ 提案する理由
・終わり時間を逆算することで生産性が上がる
・残業の常態化を防ぐことで離職リスクを下げられる
・週1回から始めることでまず効果を検証できる

◎ 補足
急ぎの仕事が入った場合は例外を認め、翌日に繰り越すルールとする。
```

## 例4：先延ばしにされがちな相手に、日時を決めさせるLINE提案文

会議で決めたことの実行が進まず、グループLINEで進捗を聞いても「確認します」「検討します」止まりになる相手向け。「◎現状の課題／◎提案する理由」のようなメール・社内文書型の階層構造ではなく、LINEでそのまま読める短さと柔らかさを優先する。

**Before（配布スキルのテンプレート通り・冷たく長い）**

```
【提案】録音分析フィードバックの運用を、本部からのアカウントログインなしで今週中に開始してください

先週開始予定でしたが、本部から各院のクラウド（Plaud）アカウントへログインするための調整を理由に、まだ止まっています。確認したところ、この調整自体が不要だと分かりましたので、そのまま今週中に始められます。

◎ 現状の課題
録音分析フィードバックは先週開始予定だったが、まだ着手できていない。
先週の会議後にLINEで進捗を確認したところ、「本部から各院のクラウドアカウントへログインするために調整中」という回答で、開始日時は未定のままだった。

◎ わかったこと（事実整理）
本部側の作業は、B4Aのフォルダから録音を取得し、プロンプトとともにAIに渡して分析するだけで完了する。Plaudアカウントへのログインは不要である。

◎ お願いしたいこと
横浜院・新宿院の対象カウンセラー1名分の直近の商談録音を使って、2026/07/10（金）17:00までに分析とフィードバックを試験実施してください。

◎ リスクと対策
アカウントログインでの調整は、情報管理を慎重に進めるためだったと思います。B4Aのフォルダ経由であれば新たなアカウント権限を増やさずに済むため、管理の面でもむしろシンプルになります。
```

このBeforeは、テンプレートの型（◎現状の課題→理由→お願い→リスク）を忠実に埋めた結果、次の問題を持ったまま出力されている。

- 「まだ着手できていない」「不要だと分かりました」など、相手の対応を暗に否定する言い回しになっている
- 日時（2026/07/10 17:00）をこちらから指定し、相手の決定権を奪っている
- 「情報管理」というリスクは的外れ（相手が本部ログインを必要と考えていた本当の理由は、知識不足や後回しであって、情報漏洩懸念ではない可能性が高い）
- 文章が長く、LINEでは読まれにくい

**After（磨き上げた模範回答）**

```
お疲れ様です。

録音分析フィードバックの件、進めていただきありがとうございます。

・録音はカウンセラーがB4Aのフォルダに上げてくれているので
　そこから取ってAIに読ませれば分析できそうです
・なのでアカウントへのログインは、たぶんなくても大丈夫そうです
・まずは横浜院と新宿院で、
　試しに動かしてみていただけると嬉しいです！

いつ頃なら試せそうでしょうか？
難しそうであれば、いつまでに動けそうか教えていただきたいです😊
```

**工夫したポイント**

- **宛名を入れない**：グループLINE全員が読むものなので、特定の個人名（「◯◯さん」）で呼びかけない。宛名を消すだけで、「2人への私信」ではなく「グループ全員への話」として自然に読める。社長・役員が見ていても違和感がない
- **否定的な言葉を避ける**：「まだ着手できていない」「不要だと分かりました」と言わず、「進めていただきありがとうございます」から入り、「たぶん大丈夫そうです」と柔らかく言い切らない表現にする
- **試す範囲を絞りすぎない**：「1名だけ試しに」だと逆に遅く見えるので、「横浜院と新宿院で」とすぐ動かせる範囲をそのまま伝える
- **日時は相手に決めさせる**：こちらから期日を指定せず「いつ頃なら試せそうでしょうか？」と聞く。ただし「難しそうであれば、いつまでに動けそうか教えていただきたいです」を必ず添えて、「検討します」だけで終わらせず、何らかの期日の言葉を引き出す設計にする
- **短く・箇条書きで**：◎の見出し階層は使わず、箇条書き3つと最後の問いかけだけにして、スマホで数秒で読める長さにする
- **語尾に表情をつける**：「〜そうです。」を三連続で並べると硬く見えるため、最後だけ「嬉しいです！」と感嘆符を入れ、絵文字（😊）を添えて、責めているのではなく前向きに聞いているトーンにする
- **的外れなリスク想定を書かない**：「情報管理」のような的外れな懸念には触れず、事実（ログイン不要そう）だけを軽く添えるにとどめる

## 例5：期日を聞いても返ってこない相手に、こちらが期日を設定して無返信でも進めるLINE提案文

例4のように「いつ頃なら試せそうでしょうか？」と相手に期日を委ねても、「確認します」のまま具体的な日付が返ってこない（または返信自体がない）ことがある。その後追い文面の例。型はSKILL.mdの「期日の決め方（二段構え）」第2段階に従う。

**Before（また返事待ちになる後追い）**

```
お疲れ様です。
先日の録音分析フィードバックの件、その後いかがでしょうか？
お忙しいところ恐れ入りますが、ご確認のほどよろしくお願いいたします。
```

丁寧だが期日の言葉がどこにもなく、「確認します」で終わる往復がもう一周増えるだけで、開始日は決まらない。

**After（磨き上げた模範回答）**

```
お疲れ様です。

録音分析フィードバックの件、こちらの準備が整いましたので、
開始日のご相談です。

・7/24（金）から、横浜院と新宿院で試験運用を始めたいと考えています
・もしこの日程でご不都合があれば、ご都合のよい別の期日をご指定ください

7/22（水）までにお返事がない場合は、
7/24（金）開始で進めさせていただきますね。
よろしくお願いします！
```

**工夫したポイント**

- **「その後いかがでしょうか？」を使わない**：相手の宿題を確認する形になり、責めるニュアンスが出るうえ、返せる言葉が結局「確認します」しかない。入り口を相手の進捗からこちらの提案（開始日）に付け替えると、相手は日程の可否だけ答えればよくなる
- **曜日まで添えて具体的に**：「7/24（金）」のように曜日を添えると、相手がカレンダーと突き合わせやすい。無理のない日付にすることで「別の期日をご指定ください」が形だけの逃げ道にならない
- **予定表現で言い切りを弱めない**：「進める予定です」「進めようと思います」だと、また確認待ちに戻る余地が残る。「進めさせていただきますね」と言い切ったうえで、語尾の「ね」や感嘆符で、最後通告ではなく前向きな確認のトーンを保つ
- **無返信＝承認の形は、相手の負担も減らす**：「問題なければ返信不要」になるため返信のハードルが下がり、決裁者が同席するグループLINEでもかえって答えやすくなる
